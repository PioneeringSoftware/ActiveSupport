1 # Active Support Kit
2 
3 [![Build Status](https://travis-ci.org/royratcliffe/ActiveSupportKit.png?branch=master)](https://travis-ci.org/royratcliffe/ActiveSupportKit)
4 
5 Active Support Kit mimics Ruby on Rails [Active Support][as]. Rails'
6 ActiveSupport gem provides a collection of utility classes and standard library
7 extensions designed primarily albeit not exclusively for working with the Rails
8 application framework. Active Support Kit's goal is to provide functionally
9 compatible classes and helper functions.
10 
11 Translating interfaces and semantics from Ruby to Objective-C is not _always_ an
12 easy task, even though most of the time it is. Although both languages share
13 some elements, e.g. dynamic typing, there remain significant differences. Ruby
14 has singleton classes, for instance, and Rails makes extensive use of them.
15 Objective-C has nothing equivalent.
16 
17 [as]:http://api.rubyonrails.org/
18 
19 The kit provides a class and a set of helper methods in its first version, as
20 follows. You can find more detailed documentation
21 [here](http://royratcliffe.github.com/ActiveSupportKit/).

50 ## Operating Systems
51 
52 The framework targets the latest versions of OS X and iOS, and depends on Foundation
53 framework's `NSJSONSerialization`, although you could easily adapt the
54 underlying requirement for JSON serialisation. Earlier versions depended on
55 [YAJL and some Objective-C wrappers](https://github.com/royratcliffe/yajl/tree/master/objc).
56 
57 ### iOS
58 
59 When building the library for iOS, remember to specify `-all_load` in
60 the Other Linker Flags (`OTHER_LDFLAGS`). You need this because the
61 library defines categories on standard NextStep foundation
62 classes. Without 'all load,' you will see run-time exceptions for
63 "selector not recognised." Linking against static libraries does not
64 automatically fix up run-time dependencies. See Apple's Technical Q&A
65 QA1490, Building Objective-C static libraries with categories, for
66 more details.
